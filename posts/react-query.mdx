**React Query(TanStack Query)** 는 서버 상태 관리에 특화된 라이브러리로, 비동기 데이터의 **fetching, caching, synchronization**을 효율적으로 처리합니다. 서버와 클라이언트의 데이터를 일관성 있게 유지하면서 코드의 간결성과 가독성을 높여줍니다.

> 클라이언트 상태(폼 입력, 토글 등)와 달리, **서버 상태**는 내가 소유하지 않고 언제든 바뀔 수 있으며 비동기적입니다. React Query는 바로 이 서버 상태를 다루기 위한 도구입니다.

## 주요 개념

### 1. Query (쿼리)

서버에서 데이터를 **읽어오는** 단위입니다. `useQuery`에 고유한 `queryKey`와 데이터를 가져오는 `queryFn`을 전달합니다. 반환값으로 `data`, `isLoading`, `isError` 등의 상태를 선언적으로 받습니다.

```tsx
import { useQuery } from "@tanstack/react-query";

function Todos() {
  const { data, isLoading, isError } = useQuery({
    queryKey: ["todos"],
    queryFn: () => fetch("/api/todos").then((res) => res.json()),
  });

  if (isLoading) return <p>불러오는 중…</p>;
  if (isError) return <p>에러가 발생했습니다.</p>;
  return (
    <ul>
      {data.map((todo) => (
        <li key={todo.id}>{todo.title}</li>
      ))}
    </ul>
  );
}
```

### 2. Mutation (뮤테이션)

서버 데이터를 **생성·수정·삭제**하는 단위입니다. `useMutation`을 사용하며, 성공 시 관련 쿼리를 무효화해 최신 데이터를 다시 가져옵니다.

```tsx
import { useMutation, useQueryClient } from "@tanstack/react-query";

function AddTodo() {
  const queryClient = useQueryClient();

  const mutation = useMutation({
    mutationFn: (title: string) =>
      fetch("/api/todos", { method: "POST", body: JSON.stringify({ title }) }),
    onSuccess: () => {
      // todos 쿼리를 무효화 → 자동 리페치
      queryClient.invalidateQueries({ queryKey: ["todos"] });
    },
  });

  return <button onClick={() => mutation.mutate("새 할 일")}>추가</button>;
}
```

### 3. Caching (캐싱)

쿼리 결과는 `queryKey`를 기준으로 캐시됩니다. 같은 키를 쓰는 컴포넌트는 캐시를 공유하므로 중복 요청이 발생하지 않습니다. `staleTime`(데이터가 신선하다고 간주되는 시간)과 `gcTime`(미사용 캐시가 메모리에서 제거되기까지의 시간)으로 동작을 조절합니다.

### 4. 자동 리페치 (Auto Refetching)

기본적으로 **창 포커스 복귀, 네트워크 재연결, 컴포넌트 마운트** 시 자동으로 데이터를 다시 가져와 최신 상태를 유지합니다. `refetchOnWindowFocus` 등 옵션으로 끌 수 있습니다.

### 5. Query Invalidation (쿼리 무효화)

`invalidateQueries`로 특정 쿼리를 "오래됨(stale)" 상태로 표시하면 React Query가 해당 데이터를 다시 가져옵니다. 위 Mutation 예제처럼 데이터 변경 후 일관성을 맞추는 핵심 수단입니다.

### 6. Devtools

`@tanstack/react-query-devtools`로 현재 캐시된 쿼리, 상태(fresh/stale/fetching), 데이터를 시각적으로 확인하며 디버깅할 수 있습니다.

## 기본 설정

앱 최상단을 `QueryClientProvider`로 감싸야 합니다.

```tsx
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";

const queryClient = new QueryClient();

export function App({ children }: { children: React.ReactNode }) {
  return <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>;
}
```

## 사용의 장점

- **자동 상태 관리**: fetching·캐싱·리페치 같은 비동기 작업을 최소한의 코드로 처리합니다.
- **데이터 일관성 보장**: 여러 컴포넌트가 같은 데이터를 공유해도 일관성이 유지됩니다.
- **유연한 캐싱·리페치 정책**: 캐시 유효 시간과 리페치 시점을 자유롭게 설정해 서버 부하를 줄입니다.
- **개발 편의성**: `useQuery`, `useMutation`으로 선언적 코드를 작성하고, DevTools로 상태를 쉽게 추적합니다.
