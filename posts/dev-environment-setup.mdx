## 1. Prettier

Prettier는 정해진 규칙에 따라 **코드 스타일을 자동으로 정리**해 주는 도구입니다.

```json
{
  "tabWidth": 2,
  "printWidth": 120,
  "singleQuote": true,
  "trailingComma": "all",
  "semi": true,
  "useTabs": false
}
```

- `tabWidth: 2` — 탭은 2 스페이스
- `printWidth: 120` — 선호하는 한 줄 길이
- `singleQuote: true` — 홑따옴표 사용
- `trailingComma: "all"` — 여러 줄로 나뉠 때 끝에 쉼표
- `semi: true` — 문장 끝 세미콜론
- `useTabs: false` — 탭 대신 스페이스

## 2. ESLint

ESLint는 자바스크립트의 **문법·품질 문제를 표시**해 주는 도구입니다.

```js
// .eslintrc.js (옛 방식)
module.exports = {
  env: { browser: true, es2021: true },
  extends: [
    "eslint:recommended",
    "plugin:react/recommended",
    "plugin:@typescript-eslint/recommended",
    "airbnb",
    "airbnb-typescript",
    "prettier",
  ],
  parser: "@typescript-eslint/parser",
  parserOptions: { ecmaFeatures: { jsx: true }, project: "./tsconfig.json" },
  plugins: ["react", "@typescript-eslint"],
  rules: {
    "react/react-in-jsx-scope": 0,
    "react/jsx-props-no-spreading": "off",
  },
};
```

설정 항목 정리:

- **env**: 프로젝트의 사용 환경(`browser` 전역 변수, `es2021` 등).
- **extends**: 공개된 설정을 기반으로 가져옵니다. 위는 Airbnb 스타일에 Prettier를 반영한 예시입니다.
- **parser**: 기본은 Espree. TypeScript는 `@typescript-eslint/parser`를 사용합니다.
- **plugins**: 기본 규칙 외 추가 규칙을 제공.
- **rules**: 세부 규칙 조정. `0`=끔, `1`=경고, `2`=에러. rules가 extends보다 우선합니다.
  - `react/react-in-jsx-scope: 0` — React 17의 새 JSX 변환부터 `import React`가 불필요해, 이 규칙을 끕니다.

> 📝 **업데이트 (ESLint 9+)**: 위 `.eslintrc` 형식은 레거시이며, 이제 **Flat Config(`eslint.config.js`)** 가 표준입니다. 배열로 설정을 합성하는 방식입니다.
>
> ```js
> // eslint.config.js
> import js from "@eslint/js";
> import tseslint from "typescript-eslint";
>
> export default [
>   js.configs.recommended,
>   ...tseslint.configs.recommended,
> ];
> ```

## 3. Vite

Vite(프랑스어로 "빠르다")는 빠르고 간결한 모던 웹 개발에 초점을 맞춘 빌드 도구입니다.

- 개발 시 **네이티브 ES Module**을 사용해, 번들링 부하를 브라우저로 넘기고 변경 파일만 빠르게 갱신합니다(빠른 HMR).
- 빌드 시에는 **Rollup** 기반으로 고도로 최적화된 정적 리소스를 생성합니다.
- 개발 서버는 **Esbuild**(Go로 작성, 매우 빠름) 기반이라 시작이 빠릅니다.

### CRA 대신 Vite를 쓰는 이유

CRA는 JavaScript로 구현된 Webpack을 사용해 코드 양이 많아질수록 느려집니다. 이를 해결하기 위해 Esbuild 기반의 Vite를 사용합니다.

> 📝 **업데이트**: **Create React App(CRA)은 공식적으로 권장이 중단**되었습니다. 새 프로젝트는 **Vite**나 **Next.js** 같은 프레임워크로 시작하는 것이 표준입니다.

## 4. Webpack과 CRA

- **Webpack**: 자바스크립트 **모듈 번들러**. 수많은 JS 파일을 모듈화해 하나로 묶어 네트워크 비용을 줄입니다. 분할된 코드를 하나의 진입점(entry)으로 모아 실행합니다.
- **CRA(Create React App)**: 페이스북이 내놓았던 공식 보일러플레이트. Webpack과 Babel로 파일 로더·빌드·ES6 지원·ESLint 등을 미리 구성해 줍니다.
- **Babel**: ES6+ 문법을 구형 브라우저용으로 변환하는 트랜스파일러. JSX 변환도 담당합니다.

---

요약하면, **Prettier**는 스타일을, **ESLint**는 품질을 책임지고, **Vite**(또는 Webpack)는 이 모든 것을 묶어 실행 가능한 결과물로 만들어 줍니다.
