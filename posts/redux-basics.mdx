Redux는 애플리케이션의 상태를 **예측 가능하게** 관리하기 위한 라이브러리입니다. 전역 상태를 한 곳에 모으고, 상태 변경을 일정한 규칙으로만 허용해 흐름을 추적하기 쉽게 만듭니다.

## 주요 개념

1. **Store(저장소)**: Redux의 중심. 애플리케이션의 모든 상태를 담으며, 한 앱에 **하나의 Store만** 존재합니다.
2. **Action(행동)**: 상태를 변경하려면 Action 객체를 디스패치합니다. Action은 `type` 속성을 필수로 가지며, 추가 데이터(payload)를 담을 수 있습니다.
3. **Reducer(리듀서)**: Action을 받아 상태를 어떻게 업데이트할지 정의하는 **순수 함수**. (이전 상태, Action) → 새로운 상태.
4. **Dispatch(디스패치)**: Action을 `dispatch` 메서드로 Store에 전달합니다. 그러면 Reducer가 상태를 업데이트합니다.
5. **Selector(선택자)**: Store에서 특정 상태를 가져오는 함수. 컴포넌트가 필요한 데이터만 선택해 사용합니다.

## 동작 원리 — 단방향 데이터 흐름

Redux는 **단방향 데이터 흐름**을 따릅니다.

1. **Dispatch**: 컴포넌트나 사용자 상호작용이 Action을 dispatch합니다.
2. **Reducer**: Action이 Reducer로 전달되고, 현재 상태와 Action에 따라 새 상태가 반환됩니다.
3. **Store 업데이트**: Store가 업데이트된 상태를 보유합니다.
4. **구독 컴포넌트 업데이트**: 변경을 구독하던 컴포넌트가 새 상태를 받아 화면을 갱신합니다.

```
Component → dispatch(action) → Reducer → Store 갱신 → 구독 컴포넌트 리렌더
```

## Redux Toolkit 예제

요즘은 보일러플레이트를 줄여주는 **Redux Toolkit(RTK)** 으로 작성하는 것이 표준입니다. `createSlice`가 action과 reducer를 한 번에 만들어 줍니다.

```ts
// store/counterSlice.ts
import { createSlice, type PayloadAction } from "@reduxjs/toolkit";

interface CounterState {
  value: number;
}

const initialState: CounterState = { value: 0 };

const counterSlice = createSlice({
  name: "counter",
  initialState,
  reducers: {
    increment: (state) => {
      // Immer 덕분에 "직접 변경"처럼 작성해도 불변성이 유지됩니다.
      state.value += 1;
    },
    incrementByAmount: (state, action: PayloadAction<number>) => {
      state.value += action.payload;
    },
  },
});

export const { increment, incrementByAmount } = counterSlice.actions;
export default counterSlice.reducer;
```

```ts
// store/index.ts
import { configureStore } from "@reduxjs/toolkit";
import counterReducer from "./counterSlice";

export const store = configureStore({
  reducer: { counter: counterReducer },
});

export type RootState = ReturnType<typeof store.getState>;
export type AppDispatch = typeof store.dispatch;
```

```tsx
// Counter.tsx
import { useSelector, useDispatch } from "react-redux";
import type { RootState } from "./store";
import { increment, incrementByAmount } from "./store/counterSlice";

export function Counter() {
  const value = useSelector((state: RootState) => state.counter.value); // Selector
  const dispatch = useDispatch();

  return (
    <div>
      <span>{value}</span>
      <button onClick={() => dispatch(increment())}>+1</button>
      <button onClick={() => dispatch(incrementByAmount(5))}>+5</button>
    </div>
  );
}
```

## Redux 사용의 장점

- **예측 가능한 상태 관리**: 상태 변화가 순수 함수인 Reducer로만 처리되어 추적·디버깅이 쉽습니다.
- **중앙 집중식 관리**: 전역 Store에서 상태를 관리해 여러 컴포넌트 간 일관성이 유지됩니다.
- **개발자 도구**: Redux DevTools로 상태 변화를 시간순으로 추적하고 분석할 수 있습니다.

> 다만 모든 상태를 Redux에 넣을 필요는 없습니다. 서버 데이터는 React Query 같은 서버 상태 라이브러리에, 지역 UI 상태는 `useState`에 두는 식으로 **상태의 성격에 맞게** 나누는 것이 좋습니다.
