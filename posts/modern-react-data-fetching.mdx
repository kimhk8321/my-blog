[(5) 커스텀 훅 편](/posts/modern-react-custom-hooks)에 이어, 실제 앱의 대부분을 차지하는 **서버 데이터 연동**을 정리합니다.

## 기본 — useEffect로 호출하기

컴포넌트가 마운트될 때 데이터를 가져오려면 `useEffect`를 씁니다. 비동기 작업은 보통 **로딩 / 성공 / 에러** 세 가지 상태로 다룹니다.

```jsx
function UserList() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    async function load() {
      try {
        setLoading(true);
        const res = await fetch("/api/users");
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        setUsers(await res.json());
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    }
    load();
  }, []);

  if (loading) return <p>로딩 중…</p>;
  if (error) return <p>에러: {error.message}</p>;
  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>{u.name}</li>
      ))}
    </ul>
  );
}
```

## 경쟁 상태(Race Condition) 처리

빠르게 바뀌는 파라미터로 연속 요청을 보내면, **늦게 도착한 옛 응답이 최신 응답을 덮어쓰는** 문제가 생길 수 있습니다. cleanup으로 오래된 요청의 결과를 무시합니다.

```jsx
useEffect(() => {
  let ignore = false;
  fetch(`/api/users/${id}`)
    .then((res) => res.json())
    .then((data) => {
      if (!ignore) setUser(data); // 최신 요청만 반영
    });
  return () => {
    ignore = true; // 이전 요청 무효화
  };
}, [id]);
```

`AbortController`로 요청 자체를 취소할 수도 있습니다.

```jsx
useEffect(() => {
  const controller = new AbortController();
  fetch(url, { signal: controller.signal }).then(/* … */);
  return () => controller.abort();
}, [url]);
```

위 패턴은 [(5)편의 `useFetch`](/posts/modern-react-custom-hooks)처럼 커스텀 훅으로 빼면 컴포넌트가 훨씬 단순해집니다.

## 직접 다루기의 한계, 그리고 React Query

`useEffect` 기반 데이터 페칭은 직접 구현하다 보면 캐싱, 중복 요청 제거, 리페치, 무효화 등 처리할 것이 계속 늘어납니다. 이런 **서버 상태 관리**는 [React Query](/posts/react-query) 같은 라이브러리에 맡기는 것이 현대적인 방식입니다.

```jsx
// React Query — 로딩/에러/캐싱이 선언적으로 해결됨
const { data, isLoading, error } = useQuery({
  queryKey: ["users"],
  queryFn: () => fetch("/api/users").then((r) => r.json()),
});
```

---

데이터 연동의 핵심은 "**로딩·에러·성공 상태를 빠짐없이 다루고, 오래된 응답을 걸러내는 것**"입니다. 직접 구현으로 원리를 이해한 뒤, 실무에서는 React Query로 옮겨 가는 흐름이 자연스럽습니다.
