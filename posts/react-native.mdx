## React Native란?

- **React 기반**: 리액트의 컴포넌트 기반 UI 설계 방식을 그대로 사용합니다.
- **네이티브 렌더링**: HTML·CSS 대신 네이티브 컴포넌트를 사용해, iOS와 Android에서 동작하는 **진정한 네이티브 앱**을 만듭니다.
- **JavaScript 활용**: 웹 React처럼 JavaScript 또는 TypeScript로 작성합니다.

## 웹 React와의 주요 차이점

| 구분 | React (웹) | React Native |
|---|---|---|
| 렌더링 대상 | DOM (브라우저) | 네이티브 UI (iOS/Android) |
| 기본 요소 | `div`, `span`, `img` … | `View`, `Text`, `Image` … |
| 스타일링 | CSS / className | `StyleSheet` 객체 + Flexbox |
| 내비게이션 | react-router 등 | React Navigation |
| 플랫폼 API | 브라우저 API | 카메라·GPS 등 네이티브 모듈 |

## 주요 컴포넌트

HTML 태그 대신 네이티브 컴포넌트를 사용합니다.

- **`View`**: `div`와 유사한 컨테이너
- **`Text`**: 텍스트 표시 (`p`, `span`과 유사)
- **`Image`**: 이미지 표시 (`img`와 유사)
- **`ScrollView`**: 스크롤 가능한 영역
- **`Touchable*` / `Pressable`**: 클릭/터치 가능한 버튼 역할

```jsx
import { View, Text, StyleSheet } from "react-native";

export default function Hello() {
  return (
    <View style={styles.box}>
      <Text style={styles.title}>안녕하세요!</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  box: { padding: 16, alignItems: "center" },
  title: { fontSize: 20, fontWeight: "bold" },
});
```

## 장점

1. **크로스 플랫폼**: 한 번의 코드 작성으로 iOS·Android 앱 개발.
2. **빠른 개발**: Fast Refresh(핫 리로딩)로 결과를 즉시 확인.
3. **네이티브 모듈 지원**: 카메라, GPS, 푸시 알림 등 네이티브 기능 연동.
4. **커뮤니티**: React 개발자라면 익숙한 생태계와 풍부한 자료를 활용.

## React 개발자의 빠른 적응법

1. 개발 환경 설치 (아래 업데이트 참고)
2. **스타일링**: `StyleSheet` 문법과 **Flexbox**에 익숙해지기 (웹과 달리 기본 `flexDirection`이 `column`).
3. **React Navigation** 학습: 네이티브 앱 라우팅 필수.
4. **디버깅**: React DevTools, 시뮬레이터 디버거 활용.
5. **시뮬레이터 세팅**: iOS/Android 플랫폼별 테스트 환경 구성.

> 📝 **업데이트**: 과거에는 `React Native CLI`로 시작하는 경우가 많았지만, 요즘은 **Expo**로 시작하는 것이 훨씬 권장됩니다. 빌드·배포·네이티브 모듈 관리가 크게 단순해집니다.
>
> ```shell
> npx create-expo-app@latest my-app
> ```

웹 React를 알고 있다면 React Native의 컴포넌트·상태·훅 개념은 그대로 가져올 수 있습니다. 차이는 주로 **렌더링 대상(네이티브 UI)** 과 **스타일링·내비게이션** 에 있습니다.
