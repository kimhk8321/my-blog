[(8) 브라우저 렌더링 편](/posts/js-deep-dive-browser-rendering)에 이어, 문자열 패턴 매칭의 강력한 도구인 **정규 표현식**을 정리합니다.

## 생성과 플래그

```javascript
const re1 = /abc/gi; // 리터럴
const re2 = new RegExp("abc", "gi"); // 생성자
```

자주 쓰는 **플래그**:

- `g` (global): 모든 일치 검색
- `i` (ignore case): 대소문자 무시
- `m` (multiline): 여러 줄 모드

## 자주 쓰는 패턴

| 패턴 | 의미 |
|---|---|
| `.` | 임의의 한 문자 |
| `\d` / `\D` | 숫자 / 숫자가 아닌 문자 |
| `\w` / `\W` | 단어 문자(영숫자_) / 그 외 |
| `\s` | 공백 |
| `^` / `$` | 문자열 시작 / 끝 |
| `*` / `+` / `?` | 0개 이상 / 1개 이상 / 0~1개 |
| `{n,m}` | n~m개 반복 |
| `[abc]` | a, b, c 중 하나 |
| `[^abc]` | a, b, c가 아닌 것 |
| `(…)` | 그룹(캡처) |
| `\|` | OR |

## 주요 메서드

```javascript
const re = /\d+/g;

re.test("abc123"); // true — 일치 여부
"a1b2c3".match(/\d/g); // ["1", "2", "3"] — 모든 일치
"2024-01-15".replace(/-/g, "/"); // "2024/01/15" — 치환
"a1b2".matchAll(/(\w)(\d)/g); // 캡처 그룹까지 반복 추출
```

캡처 그룹은 치환에서 `$1`, `$2`로 참조합니다.

```javascript
"2024-01-15".replace(/(\d{4})-(\d{2})-(\d{2})/, "$3.$2.$1");
// "15.01.2024"
```

## 실전 예제

```javascript
// 이메일 (간단 버전)
const email = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
email.test("hi@example.com"); // true

// 한국 휴대폰 번호
const phone = /^01[016789]-?\d{3,4}-?\d{4}$/;
phone.test("010-1234-5678"); // true

// 공백 제거(앞뒤 + 중복)
"  hello   world  ".trim().replace(/\s+/g, " "); // "hello world"
```

> 정규식은 강력하지만 복잡해지면 가독성이 급격히 떨어집니다. 너무 복잡한 패턴은 주석을 달거나, 차라리 코드로 풀어 쓰는 편이 유지보수에 낫습니다. 작성·테스트에는 [regex101](https://regex101.com) 같은 도구가 유용합니다.

---

정규 표현식은 폼 유효성 검사, 문자열 파싱·치환 등에서 자주 쓰입니다. 모든 패턴을 외우기보다, **자주 쓰는 메타문자와 메서드(`test`/`match`/`replace`)** 를 익혀 두고 필요할 때 조합하는 방식이 효율적입니다.
