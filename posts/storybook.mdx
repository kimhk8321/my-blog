Storybook은 UI 컴포넌트를 애플리케이션과 **독립적으로** 개발·문서화·테스트할 수 있는 도구입니다. 각 컴포넌트의 다양한 상태를 한 곳에서 확인할 수 있습니다.

## 주요 개념

1. **Stories**: UI 컴포넌트의 각 상태를 설명하는 "스토리"입니다. 예를 들어 버튼의 기본/활성/비활성 상태를 각각의 스토리로 작성합니다.
2. **Addon**: 다양한 애드온으로 기능을 확장합니다. 컴포넌트 인터랙션을 실시간으로 확인하거나, 접근성·반응형 화면을 미리 볼 수 있습니다.

## 예시 — 버튼 컴포넌트

먼저 버튼 컴포넌트를 작성합니다.

```jsx
// src/components/Button.jsx
const Button = ({ label, onClick, disabled }) => (
  <button onClick={onClick} disabled={disabled}>
    {label}
  </button>
);

export default Button;
```

이제 여러 상태를 표현하는 스토리를 작성합니다.

```jsx
// Button.stories.jsx (예전 CSF2 방식)
import Button from "../src/components/Button";

export default {
  title: "Example/Button",
  component: Button,
  argTypes: {
    label: { control: "text" },
    disabled: { control: "boolean" },
  },
};

export const Default = (args) => <Button {...args} />;
export const Disabled = (args) => <Button {...args} disabled />;
```

> 📝 **업데이트 (CSF3)**: 최신 Storybook은 스토리를 함수가 아니라 **객체**로 작성하는 CSF3 문법을 권장합니다. 더 간결하고 타입 추론도 좋습니다.
>
> ```tsx
> import type { Meta, StoryObj } from "@storybook/react";
> import Button from "../src/components/Button";
>
> const meta: Meta<typeof Button> = {
>   title: "Example/Button",
>   component: Button,
>   args: { label: "버튼" },
> };
> export default meta;
>
> type Story = StoryObj<typeof Button>;
>
> export const Default: Story = {};
> export const Disabled: Story = { args: { disabled: true } };
> ```

## 왜 쓰는가

- **독립 개발**: 페이지 전체를 띄우지 않고도 컴포넌트를 개발·확인.
- **살아있는 문서**: 디자이너·동료가 컴포넌트의 모든 상태를 한눈에 확인.
- **테스트 기반**: 인터랙션 테스트, 시각적 회귀 테스트의 토대로 활용.
