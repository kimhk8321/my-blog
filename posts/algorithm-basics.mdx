코딩 테스트에서 자주 쓰이는 알고리즘을 시간 복잡도와 함께 정리했습니다.

## 정렬 (Sorting)

| 알고리즘 | 설명 | 시간 복잡도 |
|---|---|---|
| **버블 정렬** | 인접한 두 값을 비교해 큰 값을 뒤로 보냄 | O(n²) |
| **선택 정렬** | 가장 작은 값을 찾아 앞쪽과 교환 | O(n²) |
| **삽입 정렬** | 두 번째 값부터 앞 원소들과 비교해 삽입 위치 결정 | O(n) ~ O(n²) |
| **퀵 정렬** | 분할 정복. 기준값(pivot)으로 작은/큰 값을 나눠 재귀 | 평균 O(n log n) |
| **병합 정렬** | 분해할 수 있을 만큼 나눈 뒤, 합치며 정렬 | O(n log n) |
| **셸 정렬** | 간격(k)을 줄여가며 삽입 정렬을 반복 | 약 O(n^1.5) |
| **힙 정렬** | 힙 자료구조를 이용(최대/최소 힙) | O(n log n) |

> 퀵 정렬은 평균적으로 빠르지만, 피벗 선택이 나쁘면 최악 O(n²)이 됩니다. 병합·힙 정렬은 안정적으로 O(n log n)을 보장합니다.

## 탐색 (Search)

- **이분 탐색(Binary Search)**: 정렬된 배열에서 절반씩 줄여가며 탐색. **O(log n)**
- **해시(Hash)**: 추가·삭제·조회 평균 **O(1)**
- **lower/upper bound**: 이분 탐색으로 정렬 리스트에서 "해당 값 이상/이하"의 첫 인덱스를 찾음.

## 그래프 탐색

- **DFS(깊이 우선 탐색)**: 스택 또는 재귀로 구현.
- **BFS(너비 우선 탐색)**: 큐로 구현. 최단 거리(가중치 없는 그래프)에 유용.
- **LCA(최소 공통 조상)**: Union-Find 등을 활용해 두 노드의 최초 공통 조상을 찾음.

## 동적 계획법 (DP)

**메모이제이션**으로 반복되는 연산을 줄입니다. 예를 들어 피보나치 수열을 단순 재귀로 풀면 O(2ⁿ)에 가깝지만, DP로 중간 결과를 저장하면 **O(n)** 으로 줄어듭니다.

```javascript
function fib(n) {
  const memo = [0, 1];
  for (let i = 2; i <= n; i++) memo[i] = memo[i - 1] + memo[i - 2];
  return memo[n];
}
```

## 최단 경로 — 다익스트라

특정 정점에서 다른 모든 정점으로 가는 **최단 경로**를 기록합니다. 거리 배열을 무한대로 초기화한 뒤, 가장 가까운 정점부터 최단 거리를 갱신해 나갑니다. (우선순위 큐 사용 시 O(E log V))

---

알고리즘은 "어떤 자료구조 위에서, 어떤 순서로 탐색/정렬하느냐"의 조합입니다. 각 기법의 **시간 복잡도와 적합한 상황**을 함께 기억해 두면 문제를 만났을 때 빠르게 접근법을 떠올릴 수 있습니다.
