함수형 프로그래밍(FP)은 **부수효과를 줄이고 데이터 변형을 예측 가능하게** 만드는 패러다임입니다. 리액트의 상태 불변성, 배열 메서드 체이닝 등 모던 프론트엔드 곳곳에 녹아 있습니다.

## 순수 함수 (Pure Function)

같은 입력에 항상 같은 출력을 내고, **부수효과가 없는** 함수입니다.

```javascript
// 순수: 외부에 영향 없음, 입력만으로 결과 결정
function add(a, b) {
  return a + b;
}

// 비순수: 외부 상태를 바꿈(부수효과)
let total = 0;
function addToTotal(n) {
  total += n; // 외부 변수 변경
}
```

순수 함수는 **테스트·캐싱(메모이제이션)·병렬화**가 쉽습니다.

## 불변성 (Immutability)

데이터를 직접 바꾸지 않고 **새 값을 만들어** 반환합니다. [리액트 상태 관리](/posts/modern-react-arrays-reducer)의 핵심 원칙이기도 합니다.

```javascript
// 가변(피하기): 원본 변경
arr.push(4);
obj.name = "변경";

// 불변(권장): 새 값 생성
const next = [...arr, 4];
const updated = { ...obj, name: "변경" };
```

## 일급 함수 · 고차 함수

함수를 값처럼 다룰 수 있어(일급), 함수를 인자로 받거나 반환하는 **고차 함수**를 만들 수 있습니다.

```javascript
// 고차 함수: 함수를 반환
const multiplier = (factor) => (n) => n * factor;
const double = multiplier(2);
double(5); // 10

// map/filter/reduce가 대표적인 고차 함수
[1, 2, 3].map(double); // [2, 4, 6]
```

## 함수 합성 (Composition)

작은 함수들을 조합해 큰 동작을 만듭니다.

```javascript
const compose = (f, g) => (x) => f(g(x));

const trim = (s) => s.trim();
const toUpper = (s) => s.toUpperCase();
const shout = compose(toUpper, trim);

shout("  hello  "); // "HELLO"
```

## 선언형 vs 명령형

FP는 "**어떻게**(how)"보다 "**무엇을**(what)"에 집중하는 선언형 스타일을 추구합니다.

```javascript
// 명령형
const result = [];
for (let i = 0; i < nums.length; i++) {
  if (nums[i] % 2 === 0) result.push(nums[i] * 2);
}

// 선언형 (의도가 한눈에)
const result = nums.filter((n) => n % 2 === 0).map((n) => n * 2);
```

---

함수형의 정수는 "**순수 함수 + 불변성**"입니다. 이 둘만 의식해도 코드의 예측 가능성과 테스트 용이성이 크게 올라가고, 리액트 같은 라이브러리와도 자연스럽게 맞물립니다.
