HTML은 "보이는 모양"이 아니라 **문서의 의미와 구조**를 표현하는 언어입니다. 모든 것을 `<div>`로 만드는 대신 **시맨틱 태그**를 쓰면 SEO, 접근성, 유지보수가 모두 좋아집니다.

## 시맨틱 태그란

요소의 **역할/의미**를 이름으로 드러내는 태그입니다.

```html
<!-- 의미 없는 마크업 -->
<div class="header">
  <div class="nav">…</div>
</div>
<div class="main">
  <div class="article">…</div>
</div>

<!-- 시맨틱 마크업 -->
<header>
  <nav>…</nav>
</header>
<main>
  <article>…</article>
</main>
```

주요 구조 태그: `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`.

## 왜 쓰는가

- **접근성**: 스크린 리더가 `<nav>`, `<main>` 등을 인식해 "주요 콘텐츠로 건너뛰기" 같은 탐색을 제공합니다.
- **SEO**: 검색 엔진이 문서 구조(제목 위계, 본문 영역)를 더 잘 이해합니다.
- **유지보수**: `<div class="header">`보다 `<header>`가 의도를 즉시 드러냅니다.

## 제목 위계 (h1~h6)

제목은 **순서대로, 한 단계씩** 내려가야 합니다. 크기를 키우려고 `<h3>`를 쓰는 식은 피하고, 크기는 CSS로 조절합니다.

```html
<h1>페이지 제목 (페이지당 1개 권장)</h1>
  <h2>섹션</h2>
    <h3>하위 섹션</h3>
```

## 의미를 가진 인라인 태그

- `<strong>`(중요) / `<em>`(강조): `<b>`/`<i>`는 시각적일 뿐, 의미가 필요하면 이쪽을 사용.
- `<time datetime="2026-06-28">`: 기계가 읽을 수 있는 날짜.
- `<button>` vs `<a>`: **동작은 button, 이동은 a**. `<div onclick>`은 키보드·접근성 문제를 일으킵니다.

## 폼과 레이블

입력에는 항상 `<label>`을 연결합니다. 접근성과 클릭 영역 모두에 좋습니다.

```html
<label for="email">이메일</label>
<input id="email" type="email" />
```
