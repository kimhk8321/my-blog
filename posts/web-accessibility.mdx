**웹 접근성(accessibility, a11y)** 은 장애가 있는 사용자를 포함해 **누구나** 웹을 이용할 수 있게 만드는 것입니다. 스크린 리더 사용자, 키보드만 쓰는 사용자, 저시력 사용자 등을 고려합니다.

## 1. 시맨틱 마크업이 8할

가장 효과적인 접근성 향상은 [시맨틱 HTML](/posts/html-semantic)을 쓰는 것입니다. 올바른 태그(`<button>`, `<nav>`, `<label>`)는 그 자체로 보조 기술이 이해할 수 있는 정보를 제공합니다.

```html
<!-- 나쁨: div는 키보드 포커스·역할이 없음 -->
<div onclick="submit()">제출</div>

<!-- 좋음: 포커스·엔터·역할이 기본 제공 -->
<button onClick={submit}>제출</button>
```

## 2. 대체 텍스트

이미지에는 의미를 설명하는 `alt`를 제공합니다. 장식용 이미지는 `alt=""`로 비워 스크린 리더가 건너뛰게 합니다.

```html
<img src="chart.png" alt="2026년 분기별 매출 추이" />
<img src="divider.svg" alt="" /> <!-- 장식용 -->
```

## 3. ARIA — 보충 수단

**ARIA**(Accessible Rich Internet Applications)는 시맨틱 HTML로 부족할 때 역할·상태를 보충하는 속성입니다.

```html
<button aria-label="메뉴 열기" aria-expanded="false">☰</button>
<div role="alert">저장되었습니다</div>
```

> **ARIA 제1규칙: 안 쓰는 게 최선.** 네이티브 HTML로 표현할 수 있으면 ARIA보다 그쪽을 쓰세요. 잘못 쓴 ARIA는 오히려 접근성을 해칩니다.

## 4. 키보드 내비게이션

마우스 없이 **Tab/Enter/방향키만으로** 모든 기능을 쓸 수 있어야 합니다.

- 포커스 순서가 논리적이어야 함(시각 순서와 일치).
- **포커스 표시(focus ring)를 지우지 마세요.** `outline: none`만 주는 건 금물 — 대체 스타일을 제공하거나 `:focus-visible`을 활용.
- 모달을 열면 포커스를 그 안에 가두고(focus trap), 닫으면 원래 위치로.

## 5. 색과 대비

- 텍스트와 배경의 **명도 대비**는 최소 4.5:1(WCAG AA)을 권장.
- 정보를 **색으로만** 전달하지 마세요(색맹 고려). 색 + 아이콘/텍스트를 함께.

## 점검 도구

- 브라우저 개발자도구의 **Lighthouse(Accessibility)**, **axe DevTools**
- 키보드만으로 사이트를 끝까지 써보기 / 스크린 리더(VoiceOver, NVDA)로 들어보기

---

접근성은 "추가 기능"이 아니라 **기본기**입니다. 시맨틱 마크업 + 키보드 지원 + 충분한 대비 — 이 셋만 챙겨도 대부분의 사용자가 더 편해집니다.
