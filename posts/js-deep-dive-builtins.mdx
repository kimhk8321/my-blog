## 래퍼 객체

원시값인 문자열·숫자·불리언에 점(`.`)으로 메서드를 호출할 수 있는 이유는, 자바스크립트가 그 순간 임시 **래퍼 객체**(`String`, `Number`, `Boolean`)를 만들어 주기 때문입니다.

```javascript
const str = "hello";
str.toUpperCase(); // "HELLO" — 임시 String 객체가 처리 후 사라짐
```

## Number

```javascript
(3.14159).toFixed(2); // "3.14" (반올림, 문자열 반환)
Number.parseInt("42px", 10); // 42
Number.parseFloat("3.14abc"); // 3.14
Number.isNaN(NaN); // true
Number.isInteger(42); // true
(255).toString(16); // "ff" (진법 변환)
```

## String

```javascript
const s = "  Hello, World  ";
s.trim(); // "Hello, World"
s.slice(2, 7); // "Hello"  (음수 인덱스 가능)
s.split(","); // ["  Hello", " World  "]
s.replace("Hello", "Hi"); // 첫 일치 / replaceAll로 전체
s.includes("World"); // true
"5".padStart(3, "0"); // "005"
`hi ${1 + 1}`; // 템플릿 리터럴
```

## Array

고차 함수(`map`/`filter`/`reduce`)는 [(3)편](/posts/js-deep-dive-async)에서 다뤘고, 여기서는 변형·탐색 메서드를 정리합니다.

```javascript
const arr = [1, 2, 3];
arr.push(4); // 끝에 추가 (원본 변경)
arr.pop(); // 끝 제거
arr.slice(1, 3); // [2, 3] — 원본 유지(복사)
arr.splice(1, 1, "x"); // 인덱스1부터 1개 제거 후 삽입 (원본 변경)
arr.concat([5, 6]); // 연결
[1, [2, [3]]].flat(2); // [1, 2, 3]
arr.indexOf(2); // 위치
arr.includes(2); // 포함 여부
[3, 1, 2].sort((a, b) => a - b); // 정렬 (원본 변경)
Array.from("abc"); // ["a","b","c"]
Array.isArray(arr); // true
```

> `slice`/`concat`/`map`은 **새 배열**을 반환(불변), `push`/`splice`/`sort`는 **원본을 변경**합니다. 이 구분을 알아야 상태 관리에서 버그를 피할 수 있습니다.

## Object

```javascript
const obj = { a: 1, b: 2 };
Object.keys(obj); // ["a", "b"]
Object.values(obj); // [1, 2]
Object.entries(obj); // [["a",1],["b",2]]
Object.assign({}, obj, { c: 3 }); // 병합 (또는 {...obj, c:3})
Object.freeze(obj); // 불변(수정 무시)
```

## Math · Date · JSON

```javascript
// Math
Math.max(1, 2, 3); // 3
Math.floor(4.7); // 4
Math.round(4.5); // 5
Math.random(); // 0~1 난수

// Date
const now = new Date();
now.getFullYear();
Date.now(); // 타임스탬프(ms)

// JSON
const json = JSON.stringify({ a: 1 }); // '{"a":1}'
JSON.parse(json); // { a: 1 }
```

> 날짜 계산이 복잡해지면 `date-fns`나 `Day.js` 같은 가벼운 라이브러리를 쓰는 편이 안전합니다. (표준 `Temporal` API도 도입이 진행 중입니다.)

---

표준 빌트인은 "외워야 할 목록"이라기보다, **원본을 바꾸는지/새 값을 반환하는지**의 관점으로 익히면 실수를 줄일 수 있습니다.
