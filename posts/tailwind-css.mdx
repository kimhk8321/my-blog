Tailwind CSS는 미리 정의된 **유틸리티 클래스**를 조합해 마크업에서 바로 스타일링하는 CSS 프레임워크입니다.

## 주요 개념

1. **유틸리티 클래스**: `p-4`, `text-white`처럼 하나의 속성을 담당하는 작은 클래스를 조합해 디자인합니다. 별도의 CSS 파일을 오가지 않고 마크업에서 바로 스타일을 적용합니다.
2. **모듈러 디자인**: 작은 유틸리티들의 조합으로 일관된 디자인 시스템(간격·색상·타이포)을 구성합니다. 디자인 토큰이 클래스로 표준화되어 일관성이 유지됩니다.
3. **커스터마이징**: 테마(색상·폰트·간격 등)를 프로젝트에 맞게 확장·재정의할 수 있습니다.
4. **JIT(Just-in-Time) 모드**: 실제로 사용한 클래스만 그때그때 생성합니다. 덕분에 빌드가 빠르고, 최종 CSS에 **사용하지 않는 스타일이 포함되지 않아** 번들이 작습니다.

## 사용 예시

```html
<div class="bg-blue-500 text-white font-bold p-4 rounded-lg shadow-md">
  Hello, Tailwind!
</div>
```

위 예제는 파란 배경, 흰색 텍스트, 굵은 글꼴, padding 1rem, 둥근 모서리, 그림자를 **클래스 이름만으로** 적용합니다.

## 장점과 단점

**장점**
- 빠른 스타일 적용과 디자인 일관성 유지
- 클래스명을 고민할 필요가 없고, 별도 CSS 파일 관리가 간소화
- 사용하지 않는 CSS가 제거되어 작은 번들

**단점**
- 초기 학습 곡선(클래스 이름 암기)
- 마크업에 클래스가 길게 붙어 가독성이 떨어질 수 있음 (컴포넌트 추출로 완화)

> 📝 **업데이트 (Tailwind v4)**: v4부터는 **CSS-우선 설정**으로 바뀌어, `tailwind.config.js` 대신 CSS 파일에서 `@import "tailwindcss";` 와 `@theme` 로 설정합니다. 또한 JIT가 기본 동작이며, 빌드 엔진(Oxide)이 Rust로 재작성되어 더 빨라졌습니다.
>
> ```css
> @import "tailwindcss";
>
> @theme {
>   --color-brand: #4f46e5;
> }
> ```
