## 커스텀 훅이란

여러 컴포넌트에서 반복되는 **상태 관련 로직**을 `use`로 시작하는 함수로 분리한 것입니다. 내부에서 `useState`, `useEffect` 등 기본 훅을 조합합니다. [HOC나 렌더 프롭](/posts/design-patterns)보다 훨씬 간결하게 로직을 공유합니다.

**규칙**

- 이름은 반드시 **`use`로 시작**합니다(리액트가 훅으로 인식).
- 컴포넌트 최상위에서만 호출합니다(조건문·반복문 안에서 호출 금지).

## 예제 1 — useToggle

```jsx
import { useState, useCallback } from "react";

function useToggle(initial = false) {
  const [value, setValue] = useState(initial);
  const toggle = useCallback(() => setValue((v) => !v), []);
  return [value, toggle];
}

// 사용
function Modal() {
  const [isOpen, toggle] = useToggle();
  return <button onClick={toggle}>{isOpen ? "닫기" : "열기"}</button>;
}
```

## 예제 2 — useFetch (데이터 로딩)

```jsx
import { useState, useEffect } from "react";

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let ignore = false; // 경쟁 상태(race condition) 방지
    setLoading(true);
    fetch(url)
      .then((res) => res.json())
      .then((json) => !ignore && setData(json))
      .catch((err) => !ignore && setError(err))
      .finally(() => !ignore && setLoading(false));
    return () => { ignore = true; };
  }, [url]);

  return { data, loading, error };
}

// 사용 — 컴포넌트가 매우 단순해진다
function UserProfile({ id }) {
  const { data, loading, error } = useFetch(`/api/users/${id}`);
  if (loading) return <p>로딩 중…</p>;
  if (error) return <p>에러 발생</p>;
  return <h1>{data.name}</h1>;
}
```

## 예제 3 — useDebounce

검색 입력처럼 값이 자주 바뀔 때, 일정 시간 멈춘 뒤의 값만 반영합니다.

```jsx
import { useState, useEffect } from "react";

function useDebounce(value, delay = 300) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer); // 입력이 이어지면 이전 타이머 취소
  }, [value, delay]);
  return debounced;
}

// 사용
function Search() {
  const [keyword, setKeyword] = useState("");
  const debouncedKeyword = useDebounce(keyword, 500);
  // debouncedKeyword가 바뀔 때만 API 호출…
  return <input value={keyword} onChange={(e) => setKeyword(e.target.value)} />;
}
```

---

커스텀 훅의 가치는 "**로직과 UI의 분리**"입니다. 컴포넌트는 화면에 집중하고, 상태 로직은 훅으로 빼내면 재사용성과 테스트 용이성이 크게 올라갑니다. 좋은 리액트 코드는 결국 잘 만든 커스텀 훅의 모음인 경우가 많습니다.
