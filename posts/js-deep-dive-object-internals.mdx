## 프로퍼티 어트리뷰트

자바스크립트 엔진은 프로퍼티를 생성할 때 **프로퍼티 어트리뷰트**라는 내부 상태를 함께 정의합니다. `Object.getOwnPropertyDescriptor`로 확인할 수 있습니다.

```javascript
const person = { name: "형기" };
Object.getOwnPropertyDescriptor(person, "name");
// { value: "형기", writable: true, enumerable: true, configurable: true }
```

- **value**: 프로퍼티 값
- **writable**: 값 변경 가능 여부
- **enumerable**: 열거(`for…in`, `Object.keys`) 가능 여부
- **configurable**: 재정의/삭제 가능 여부

`Object.defineProperty`로 어트리뷰트를 직접 정의할 수 있습니다.

```javascript
const obj = {};
Object.defineProperty(obj, "id", {
  value: 1,
  writable: false, // 읽기 전용
  enumerable: false, // 열거되지 않음
});
obj.id = 2; // 무시됨 (strict 모드면 에러)
```

## 데이터 프로퍼티 vs 접근자 프로퍼티

- **데이터 프로퍼티**: 값을 가지는 일반 프로퍼티(`value`, `writable`…).
- **접근자 프로퍼티**: 자체 값 없이 `get`/`set` 함수로 값을 읽고 쓰는 프로퍼티.

```javascript
const user = {
  firstName: "형기",
  lastName: "김",
  get fullName() {
    return `${this.lastName}${this.firstName}`;
  },
  set fullName(name) {
    [this.lastName, this.firstName] = name.split(" ");
  },
};

user.fullName; // "김형기" (getter)
user.fullName = "이 영희"; // setter
```

## 생성자 함수에 의한 객체 생성

객체 리터럴은 하나의 객체만 만들지만, **생성자 함수**는 같은 구조의 객체를 여러 개 찍어낼 수 있습니다(템플릿 역할).

```javascript
function Circle(radius) {
  this.radius = radius;
  this.getArea = function () {
    return Math.PI * this.radius ** 2;
  };
}

const c1 = new Circle(1);
const c2 = new Circle(2);
```

`new` 연산자와 함께 호출하면 ① 빈 객체 생성 → ② `this` 바인딩 → ③ 프로퍼티 추가 → ④ 객체 반환이 일어납니다.

> 위처럼 메서드를 생성자 안에서 정의하면 인스턴스마다 함수가 중복 생성됩니다. **메서드는 프로토타입에 정의**하면 모든 인스턴스가 공유해 메모리를 아낄 수 있습니다.
>
> ```javascript
> Circle.prototype.getArea = function () {
>   return Math.PI * this.radius ** 2;
> };
> ```

## 불변 객체

```javascript
Object.freeze(obj); // 동결: 추가·삭제·수정 모두 불가
Object.isFrozen(obj); // true
Object.seal(obj); // 밀봉: 추가·삭제 불가, 수정은 가능
```

단, `freeze`는 **얕은 동결**입니다. 중첩 객체까지 막으려면 재귀적으로 동결(deep freeze)해야 합니다.
