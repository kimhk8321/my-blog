코드는 한 번 쓰이고 **여러 번 읽힙니다.** 클린 코드는 "사람이 읽기 좋은 코드"를 지향하고, 리팩터링은 "동작을 바꾸지 않고 구조를 개선"하는 작업입니다.

## 네이밍 — 의도를 드러내기

```javascript
// 나쁨
const d = 30; // 무슨 값?
const list = users.filter((u) => u.a);

// 좋음
const maxRetryCount = 30;
const activeUsers = users.filter((user) => user.isActive);
```

- 이름만으로 **무엇/왜**가 드러나게.
- 불리언은 `is`/`has`/`can` 접두사.
- 줄임말·매직 넘버를 피하고 상수로 명명.

## 함수 — 작게, 한 가지만

- 함수는 **하나의 일**만 한다([단일 책임](/posts/oop-solid)).
- 매개변수는 적게(3개 이하 권장). 많아지면 객체로 묶기.
- 부수효과를 함수 이름에 드러내거나 분리.

```javascript
// 나쁨: 검증 + 저장 + 알림이 뒤섞임
function register(user) { /* 50줄 */ }

// 좋음: 작은 함수로 분리
function register(user) {
  validate(user);
  save(user);
  notify(user);
}
```

## 주석보다 코드로

좋은 코드는 **주석 없이도 읽힙니다.** 주석이 필요하다면, 먼저 코드를 더 명확하게 만들 수 없는지 봅니다. 다만 "**왜** 이렇게 했는지"(의도·배경)는 주석으로 남길 가치가 있습니다.

```javascript
// 나쁨: 코드를 그대로 설명 (불필요)
i = i + 1; // i를 1 증가

// 좋음: 비직관적 결정의 '이유'
// 외부 API가 1-base 인덱스를 쓰므로 +1 보정
page = page + 1;
```

## 코드 스멜 (리팩터링 신호)

- **중복 코드**: 같은 로직이 여러 곳 → 함수/훅으로 추출
- **긴 함수 / 큰 컴포넌트**: 분리
- **깊은 중첩(if/for)**: 조기 반환(early return), 가드 절로 평탄화
- **매직 넘버/문자열**: 상수화

```javascript
// 깊은 중첩
function f(user) {
  if (user) {
    if (user.isActive) {
      return doSomething(user);
    }
  }
  return null;
}

// 가드 절로 평탄화
function f(user) {
  if (!user || !user.isActive) return null;
  return doSomething(user);
}
```

## 안전한 리팩터링

리팩터링의 전제는 "**동작을 바꾸지 않는다**"입니다. 그래서 [테스트](/posts/react-testing)가 뒷받침되어야 안심하고 구조를 바꿀 수 있습니다.

1. 테스트로 현재 동작을 고정
2. 작은 단위로 변경
3. 매 단계마다 테스트 통과 확인

---

클린 코드의 핵심은 거창한 규칙이 아니라 "**다음에 읽을 사람(미래의 나 포함)을 배려한다**"는 태도입니다. 명확한 이름과 작은 함수, 그리고 테스트가 받쳐주는 리팩터링이면 충분합니다.
